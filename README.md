# 星轨平台 MVP

星轨平台 MVP 用于验证学生发现竞赛/科研机会、发布组队招募、申请科研项目和维护证书记录的核心流程。

## 项目结构

- `public/`: 静态页面和样式
- `src/`: MVP 领域逻辑和浏览器交互
- `tests/`: 核心流程行为测试
- `scripts/`: 本地启动脚本
- `MVP/`: 课程要求的优先级清单和验证反馈报告
- `CONTEXT.md`: 项目领域词汇表

## 环境要求

- Node.js 20 或更高版本
- npm 10 或更高版本

## 运行

在仓库根目录执行：

```bash
npm install
npm start
```

启动后访问：

```text
http://localhost:4173
```

如需换端口：

```bash
PORT=4300 npm start
```

Windows PowerShell:

```powershell
$env:PORT=4300; npm start
```

## 测试

在仓库根目录执行：

```bash
npm test
```

测试覆盖：

- 学生注册与登录
- 竞赛/科研机会浏览与筛选
- 组队招募发布与竞赛详情展示
- 科研项目申请与导师侧查看
- 证书记录上传校验与列表展示

## MVP 范围

已完成：

- 学生本地注册登录
- 注册时选择权限身份：学生、导师、证书收集者、管理员
- 机会大厅
- 竞赛组队招募
- 科研申请、导师发布/删除科研项目、导师审批申请
- 学生可查看科研申请结果；通过后显示后续联系方式，未通过显示导师反馈
- 学生可删除自己的科研申请和证书记录
- 独立证书中心，支持图片证书预览、学生证书记录保存、证书收集者汇总查看
- 证书收集者可按竞赛折叠查看、下载单份证书，并一键导出竞赛证书 zip 合集
- 管理员模式，可查看注册用户、可读数据库视图，并删除竞赛、科研项目和测试记录
- 管理员可新增竞赛和科研项目，新增内容会进入机会大厅
- 使用信息记录与手动反馈收集
- 本地数据库文件持久化，运行本地服务时数据保存到 `data/state.local.json`

暂不包含：

- 真实后端数据库
- 邮箱验证
- 真实文件上传到对象存储
- PDF 证书缩略图渲染
- 身份真实性核验

## 数据说明

运行 `npm start` 时，MVP 会通过本地服务读写 `data/state.local.json`。整个 `data/` 目录是本机演示数据目录，已加入 `.gitignore`，不会提交到 GitHub。

页面仍会把数据同步到浏览器 `localStorage` 作为兜底。需要清空数据时，可删除 `data/state.local.json`，也可在浏览器开发者工具中删除 `xinggui_mvp_state_v1` 和 `xinggui_mvp_session_v1`。
