# 星轨平台

星轨平台帮助学生发现竞赛与科研机会，并围绕这些机会完成组队招募、申请和成果证书管理。

## Language

**机会**:
学生可以浏览并进一步参与的竞赛或科研项目。
_Avoid_: 项目（单独使用时容易混淆竞赛与科研项目）

**竞赛**:
面向学生报名参赛的信息条目，包含级别、时间、官网和交流群。
_Avoid_: 比赛项目

**科研项目**:
由导师发布、面向学生招募参与者的研究机会。
_Avoid_: 科研、课题（单独使用时含义过宽）

**组队招募**:
学生围绕某个竞赛发起的队友招募信息。
_Avoid_: 招募、帖子

**申请**:
学生对某个组队招募或科研项目表达参与意向的记录。
_Avoid_: 报名（仅用于竞赛官方流程时使用）

**证书记录**:
学生保存的获奖证书元数据和文件引用。
_Avoid_: 证书（容易混淆实体文件与平台记录）

**证书收集者**:
查看学生提交的证书记录并用于汇总材料的用户。
_Avoid_: 管理员（权限范围过宽）

**管理员**:
在 MVP 演示中查看注册用户、检查数据库快照并清理测试数据的用户。
_Avoid_: 超级用户

**数据库快照**:
管理员看到的当前演示数据表及其记录集合。
_Avoid_: 后台数据（含义过宽）

**本地数据库文件**:
MVP 本地服务读写的 JSON 状态文件，用于保存演示数据并方便管理员检查。
_Avoid_: 真实数据库（容易误解为生产级后端）

**使用反馈**:
早期用户在试用 MVP 时留下的使用信息和主观意见记录。
_Avoid_: 问卷（只是一种收集形式）

**导师**:
发布科研项目并审核学生申请的教师用户。
_Avoid_: 老师

**学生**:
浏览机会、发起组队招募、提交申请并维护证书记录的用户。
_Avoid_: 用户（讨论业务行为时过宽）

## Relationships

- A **竞赛** can have zero or more **组队招募**.
- A **学生** can publish zero or more **组队招募**.
- A **科研项目** belongs to exactly one **导师**.
- A **导师** can create, delete, and review applications for their own **科研项目**.
- A **学生** can submit zero or more **申请**.
- An **申请** targets exactly one **组队招募** or one **科研项目**.
- A **学生** can own zero or more **证书记录**.
- A **证书收集者** can view many **证书记录**.
- A **使用反馈** can be linked to a **学生**, **导师**, or **证书收集者**.
- An **管理员** can inspect a **数据库快照** and delete selected test records.
- A **本地数据库文件** stores the current MVP state when the local service is running.

## Example Dialogue

> **Dev:** "学生在首页看到的是项目列表吗？"
> **Domain expert:** "用 **机会** 更准确，因为列表里同时有 **竞赛** 和 **科研项目**。学生点进 **竞赛** 后可以看或发布 **组队招募**，点进 **科研项目** 后可以提交 **申请**。"

## Flagged Ambiguities

- "项目"在需求文档中同时指 **竞赛** 和 **科研项目**; resolved: 面向学生浏览的聚合列表称为 **机会**，具体条目分别称为 **竞赛** 或 **科研项目**.
- "证书"可能指实体文件或平台中的数据; resolved: 平台持久化的数据称为 **证书记录**.
- "管理员"在证书场景中过宽; resolved: MVP 中查看证书汇总的人称为 **证书收集者**.
