# 星轨平台 MVP 验证与反馈报告

## 验证结论

当前 MVP 已跑通以下核心流程：

- 学生注册并登录。
- 学生在机会大厅浏览竞赛与科研项目，并按类型筛选。
- 学生围绕竞赛发布组队招募，招募显示在竞赛详情中。
- 学生申请科研项目，导师侧演示面板可查看待审核申请。
- 学生在证书中心保存证书记录，系统校验证书文件格式和大小，图片型证书可显示预览，证书收集者可查看汇总。
- 用户可留下联系方式、痛点、评分和手动反馈，系统会记录基础使用动作。
- 管理员可查看注册用户、可读数据库视图，并清理竞赛、科研项目和测试记录。
- 管理员可新增竞赛和科研项目，新增内容会进入机会大厅。
- 运行本地服务时，演示状态会写入 `data/state.local.json`，便于检查和复用。

- `certificate collector can review readable records from uploaders`

- `admin usage records explain actions, users, and targets in readable language`
- `admin can add competitions and research projects for the opportunity hall`
- `early user usage information and manual feedback can be collected`
- `local database stores MVP state in a readable data folder json file`

## 用户痛点解决情况分析

| 痛点 | MVP 对应能力 | 验证方式 | 当前判断 |
| --- | --- | --- | --- |
| 竞赛与科研信息分散 | 机会大厅聚合展示 | 观察用户是否能在 1 分钟内找到目标机会 | 待真实试用确认 |
| 找队友信息沉没快 | 竞赛详情中的组队招募卡片 | 观察用户是否能完成招募发布并复查展示结果 | 待真实试用确认 |
| 科研申请缺少统一入口 | 科研申请表与导师侧申请队列 | 观察学生提交后导师侧是否能看到记录 | 待真实试用确认 |
| 证书材料整理麻烦 | 独立证书中心、图片预览、上传者证书记录列表、收集者证书汇总 | 观察上传者是否看得懂自己的记录，收集者是否能切换身份并看到汇总 | 待真实试用确认 |
| 试用数据难清理和维护 | 管理员模式、数据库快照、本地数据库文件、新增机会表单 | 观察管理员是否能找到测试记录、删除记录、补充竞赛与科研项目 | 待真实试用确认 |
| 真实反馈难沉淀 | 使用反馈表和使用动作记录 | 观察用户是否愿意留下联系方式和痛点描述 | 待真实试用确认 |

## 风险与限制

- 当前数据保存在本地 `data/state.local.json` 和浏览器 `localStorage`，换设备不会同步。
- 当前权限身份由用户自行选择，尚未核验身份真实性。
- 当前证书文件只做格式与大小校验，不会上传真实文件内容。
- 真实用户反馈和联系方式需要团队在线下或问卷中补充。
