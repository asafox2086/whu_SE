# 星轨平台 MVP 验证与反馈报告

## 验证结论

当前 MVP 已跑通以下核心流程：

- 学生注册并登录。
- 学生在机会大厅浏览竞赛与科研项目，并按类型筛选。
- 学生围绕竞赛发布组队招募，招募显示在竞赛详情中。
- 学生申请科研项目，导师侧演示面板可查看待审核申请。
- 学生保存证书记录，系统校验证书文件格式和大小。

## 自动化验证

运行命令：

```bash
cd code
npm test
```

已覆盖的行为测试：

- `student can register and then log in`
- `student can browse opportunities across competitions and research projects`
- `student can publish a team recruit for an existing competition`
- `student can apply to an open research project and mentor can review it`
- `student can upload a valid certificate record and list it later`

## 用户痛点解决情况分析

| 痛点 | MVP 对应能力 | 验证方式 | 当前判断 |
| --- | --- | --- | --- |
| 竞赛与科研信息分散 | 机会大厅聚合展示 | 观察用户是否能在 1 分钟内找到目标机会 | 待真实试用确认 |
| 找队友信息沉没快 | 竞赛详情中的组队招募卡片 | 观察用户是否能完成招募发布并复查展示结果 | 待真实试用确认 |
| 科研申请缺少统一入口 | 科研申请表与导师侧申请队列 | 观察学生提交后导师侧是否能看到记录 | 待真实试用确认 |
| 证书材料整理麻烦 | 证书记录列表 | 观察用户是否理解文件格式和大小限制 | 待真实试用确认 |

## 迭代计划

| 版本 | 目标 | 计划功能 |
| --- | --- | --- |
| v0.2 | 补齐申请闭环 | 导师审核通过/拒绝、学生查看申请状态、组队招募申请加入 |
| v0.3 | 接入真实数据 | 管理员维护竞赛、导师发布科研项目、数据持久化 |
| v0.4 | 提升材料管理 | 对象存储上传、证书下载、简历与证书关联 |

## 风险与限制

- 当前数据保存在浏览器 `localStorage`，刷新仍保留，但换设备不会同步。
- 当前导师侧是演示面板，尚未实现导师登录和权限隔离。
- 当前证书文件只做格式与大小校验，不会上传真实文件内容。
- 真实用户反馈和联系方式需要团队在线下或问卷中补充。
