# 星轨平台 MVP 需求优先级清单

## MVP 目标

验证学生能否在一个入口发现竞赛与科研机会，并围绕机会完成组队招募、科研申请和证书记录维护。

## P0 必须完成

| 编号 | 需求 | 用户痛点 | 验收标准 | 当前状态 |
| --- | --- | --- | --- | --- |
| P0-1 | 学生注册与登录 | 学生需要保存自己的操作记录 | 学生可注册账号并用邮箱密码登录 | 已完成 |
| P0-2 | 机会大厅 | 竞赛和科研信息分散，学生难以快速发现 | 首页展示竞赛与科研项目，可按类型筛选 | 已完成 |
| P0-3 | 竞赛组队招募 | 学生找队友依赖群聊刷屏，信息沉没快 | 学生可针对竞赛发布组队招募，竞赛详情展示招募卡片 | 已完成 |
| P0-4 | 科研项目申请 | 学生不知道如何表达科研参与意向，导师难集中查看 | 学生可提交科研申请，导师侧可查看待审核队列 | 已完成 |
| P0-5 | 证书记录 | 学生获奖材料分散，复用和整理成本高 | 学生可保存证书记录，限制 PDF/JPG/PNG 且不超过 5MB | 已完成 |
| P0-6 | 核心流程测试 | MVP 需要避免重大致命 Bug | 注册登录、机会浏览、招募、科研申请、证书记录均有自动化测试 | 已完成 |
| P0-7 | 使用反馈收集 | MVP 需要可追踪早期用户试用情况 | 记录用户使用动作，允许用户留下联系方式、痛点、评分和手动反馈 | 已完成 |
| P0-8 | 权限身份选择 | 不同试用者需要看到不同操作入口 | 注册时可选择学生、导师、证书收集者或管理员，现阶段不核验身份真实性 | 已完成 |
| P0-9 | 管理员模式 | 演示和试用过程中需要清理测试数据 | 管理员可查看注册用户、数据库快照，并删除证书记录、反馈、申请等测试记录 | 已完成 |
| P0-10 | 证书中心与图片预览 | 只显示文件名不利于确认上传材料 | 证书有独立栏目，图片型证书显示预览图 | 已完成 |

## P1 下一轮优先

| 编号 | 需求 | 原因 | 暂缓原因 |
| --- | --- | --- | --- |
| P1-1 | 导师审核通过/拒绝申请 | 完成科研申请闭环 | MVP 先验证提交与集中查看 |
| P1-2 | 组队招募申请加入 | 完成队友双向选择 | MVP 先验证招募发布和曝光 |
| P1-3 | 管理员维护竞赛信息 | 保证机会数据持续更新 | 当前用内置样例数据完成流程验证 |
| P1-4 | 真实文件上传与对象存储 | 支持生产级证书文件管理 | MVP 先验证格式、大小、图片预览、上传者记录与收集者汇总 |

## P2 暂不纳入 MVP

| 编号 | 需求 | 暂不纳入原因 |
| --- | --- | --- |
| P2-1 | 邮箱验证、找回密码、封禁日志 | 认证增强，不影响核心价值验证 |
| P2-2 | GitHub API 集成 | 可作为简历增强，但不是机会流转必需 |
| P2-3 | 多角色权限后台 | 会显著增加实现成本，MVP 用导师侧演示面板验证 |
| P2-4 | 数据看板与推荐算法 | 属于优化功能，需先有真实使用数据 |

## 需求变更应对

- 若反馈显示学生最痛的是“找队友”，下一轮优先补 `组队招募申请加入` 和队长审核。
- 若反馈显示导师端价值更高，下一轮优先补 `导师审核通过/拒绝申请` 和导师发布科研项目。
- 若反馈显示证书管理低频，后续版本可推迟真实文件上传，把资源投入机会与申请闭环。
